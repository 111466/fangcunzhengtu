# Tiny Swords 塔防项目开发进度与迭代计划

更新日期：2026-05-05

---

## 一、当前已完成内容

本轮已经根据 `Tiny_Swords_塔防游戏开发指南.md` 的阶段 1 和阶段 2 的目标，先完成了一个可运行的塔防核心逻辑版本，并用线框和色块作为临时美术占位。

### 1. 已完成的模块拆分

当前 `scripts/` 目录已经拆分为以下模块：

- `main.lua`
  - 负责游戏状态管理、输入处理、主循环更新、渲染调度
- `Config.lua`
  - 保存世界尺寸、初始金币/生命、建造点、基础颜色配置
- `Utils.lua`
  - 通用数学和坐标转换工具
- `Path.lua`
  - 管理敌人路径点、路径长度计算、路径绘制
- `Enemy.lua`
  - 管理敌人类型、移动、受伤、减速、逃脱与绘制
- `Tower.lua`
  - 管理塔类型、索敌、攻击、升级与绘制
- `Projectile.lua`
  - 管理投射物飞行、命中、减速、溅射与绘制
- `WaveManager.lua`
  - 管理波次、准备时间、刷怪队列与胜利条件
- `UI.lua`
  - 管理顶部 HUD、选择提示、开始界面和结算界面

### 2. 已完成的游戏功能

- 已实现基础游戏状态：
  - `menu`
  - `playing`
  - `paused`
  - `game_over`
  - `victory`

- 已实现基础塔防闭环：
  - 敌人沿固定路径移动
  - 玩家在固定建造点放置防御塔
  - 防御塔自动索敌并攻击
  - 子弹追踪目标并命中
  - 击杀敌人获得金币
  - 敌人到达终点扣除生命
  - 所有波次完成后判定胜利

- 已实现 3 种敌人：
  - `grunt` 普通敌人
  - `scout` 快速敌人
  - `tank` 重甲敌人

- 已实现 3 种塔：
  - `archer` 弓箭塔，标准远程输出
  - `warrior` 战士塔，近距离高伤害并带溅射
  - `monk` 僧侣塔，远程减速控制

- 已实现 5 波敌人配置：
  - 具有波次准备时间
  - 支持混合敌人刷出
  - 波次结束后自动进入下一轮准备

### 3. 已完成的占位表现

当前尚未正式接入 Tiny Swords 精灵图，使用的是以下占位方案：

- 路径：粗线条路径段
- 建造点：圆形线框标记
- 防御塔：方形/菱形线框单位
- 敌人：方块占位并带血条
- 子弹：简单圆点
- 攻击范围：选中塔后显示圆形范围线

## 二、当前交互说明

### 1. 已支持的操作

- `Enter` / 点击屏幕：开始游戏
- `1` / `2` / `3`：切换当前准备放置的塔类型
- 左键点击建造点：放置塔
- 左键点击已建造的塔：选中塔
- `U`：升级当前选中的塔
- `P`：暂停 / 继续
- `R`：重新开始
- `Esc`：返回开始界面

### 2. 当前设计取舍

为了先把主逻辑跑通，这一版采用了下面的简化设计：

- 建塔方式为固定建造点，不是自由网格放置
- 地图路径使用手写路径点，尚未从 `assets/map.json` 自动读取
- 目前没有正式特效、音效和精灵动画
- 没有对象池优化，敌人和投射物仍是直接创建和销毁
- 没有数值平衡，当前参数仅用于验证玩法闭环

## 三、当前已知问题与风险

### 1. 需要尽快验证的内容

- 需要在 UrhoX 运行环境中实际测试鼠标点击和触屏事件是否都正常
- 需要确认 `urhox-libs/UI` 在当前工程中对绝对定位和遮罩显示是否稳定
- 需要确认 NanoVG 绘制层与 UI 层在不同分辨率下是否都正常

### 2. 当前逻辑层面的不足

- 路径和建造点还没有与 `assets/map.json` 地图真正对齐
- 没有塔出售、暂停菜单、倍速、下一波按钮等操作功能
- 没有敌人特殊技能、塔分支升级、Boss 机制
- 还没有完整的经济节奏设计
- 暂时没有测试脚本和自动化验证

### 3. 后续替换素材时的注意点

- 需要先确定 Tiny Swords 中每类单位的具体素材来源
- 需要统一资源命名和加载策略，避免后面替换时大面积改逻辑
- 替换美术时应尽量只改渲染层，不改玩法逻辑层

## 四、后续迭代总目标

后续建议按照“先稳逻辑，再贴地图，再换素材，最后做打磨”的顺序推进：

1. 先完成运行联调和逻辑修正
2. 再让地图、路径和建造点与 `map.json` 对齐
3. 再逐步替换为 Tiny Swords 正式素材
4. 最后补齐特效、音效、平衡和移动端适配

## 五、建议的迭代阶段

### 迭代 A：联调与稳定化

目标：确认当前逻辑版在工程里稳定可玩

建议任务：

- 启动项目并验证完整流程
- 检查点击放塔、升级、暂停、重开是否正常
- 检查不同窗口尺寸下 UI 和 NanoVG 的适配
- 修正输入事件字段、布局偏差、状态切换 bug
- 补充必要的调试输出和临时辅助信息

完成标志：

- 能稳定从开始界面进入战斗并完成至少 3 波
- 不出现明显的输入失效、界面错位或崩溃问题

### 迭代 B：地图对齐与场景结构

目标：让游戏逻辑与当前项目已有地图资源对齐

建议任务：

- 读取并分析 `assets/map.json`
- 根据地图重新确定世界尺寸、路径走向和建造点位置
- 如有需要，加入地图底图或简化地图渲染
- 保证路径、敌人和建造点在视觉上与地图一致

完成标志：

- 不再使用纯手写的临时路径布局
- 当前地图资源可以直接支撑核心玩法

### 迭代 C：正式美术替换

目标：将线框占位替换为 Tiny Swords 素材

建议任务：

- 为 3 种塔绑定正式单位或建筑素材
- 为 3 种敌人绑定正式敌方素材
- 给路径、草地、水域和建造点增加正式表现
- 为投射物增加基础贴图或更明确的视觉样式
- 调整锚点、缩放、朝向和层级

完成标志：

- 主要游戏元素不再依赖线框占位
- 画面能体现 Tiny Swords 的整体风格

### 迭代 D：表现增强

目标：提升打击感和反馈

建议任务：

- 增加命中特效、死亡特效、减速特效
- 增加建塔和升级反馈
- 增加简单音效
- 增强 HUD 层次和信息可读性
- 视情况加入波次开始提示和危险提示

完成标志：

- 玩家能明显感受到攻击、击杀、升级和受损反馈

### 迭代 E：玩法扩展与数值平衡

目标：把逻辑验证版推进到更完整的可玩版本

建议任务：

- 扩展更多波次
- 调整金币、伤害、射速、血量、奖励等数值
- 增加 Boss 波或精英敌人
- 增加塔的分支升级或额外技能
- 考虑加入出售、防线信息、战斗倍速等功能

完成标志：

- 至少有一套完整、可通关且有节奏变化的关卡流程

### 迭代 F：发布前打磨

目标：为后续提交或展示做准备

建议任务：

- 整理 `game.json`
- 补充 README 和操作说明
- 做移动端触屏体验优化
- 检查性能热点，必要时引入对象池
- 清理无用调试代码和占位资源

完成标志：

- 具备一个可演示、可交接、可继续扩展的稳定版本

- 可以继续作为正式塔防项目的基础框架
